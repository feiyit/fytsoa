# 基础资料（供应商/仓库/库位/地点/分类）

基础资料用于“统一口径”，避免资产与单据里出现大量手工输入、写法不一致的问题。建议先配置好基础资料，再开始录入资产与业务单据。

## 1) 供应商（Vendor）

用途：
- 资产采购来源、维修供应商
- 统计“供应商维度”的资产数量/维修费用/质保到期

建议维护字段（按你们实际页面为准）：
- 供应商名称（必填）
- 联系人/电话/地址（可选）
- 备注（可选）

常见用法：
- 资产台账中选择“供应商”
- 维修工单中选择“维修供应商”
- 入库类单据记录“供应商”

## 2) 仓库与库位（Warehouse / WarehouseBin）

用途：
- 资产的“存放组织维度”（仓库）与“更精细的位置”（库位）
- 调拨、入库、出库、盘点等业务需要明确仓库/库位

建议做法：
- 仓库：按物理地点或管理边界建立（如：总部仓、华东仓、备件仓）
- 库位：按仓库内部货架/区域建立（如：A区-01-02）

常见用法：
- 资产台账记录：当前仓库/库位
- 单据管理记录：来源仓库/目标仓库（以及行级的仓库/库位/地点覆盖）
- 盘点计划：范围可能包含仓库/库位/地点

## 3) 地点（Location）

用途：
- 资产的“物理地点/安装地点/使用地点”
- 与仓库/库位的区别：仓库/库位偏“库存管理”，地点偏“实际使用位置”

建议做法：
- 地点树按“园区/楼栋/楼层/房间”建立

常见用法：
- 资产台账记录：当前地点
- 盘点明细对比：系统地点 vs 实际地点

## 4) 资产分类（Category）

用途：
- 对资产进行归类（如：IT设备/办公家具/车辆/工装夹具）
- 列表筛选、统计报表、折旧策略（可按分类做默认策略）

实现说明（给维护人员）：
- 分类通常复用系统字典（`sys_code`），类型建议为 `AM_ASSET_CATEGORY`

常见用法：
- 资产台账：必选
- 折旧配置：可按分类制定默认折旧参数（若系统实现了联动）
- 自定义字段：可按分类定义“扩展字段模板”（若系统实现了动态表单）
