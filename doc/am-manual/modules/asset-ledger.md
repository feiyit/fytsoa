# 资产台账（资产主数据）

资产台账是系统的核心：每一条资产都是后续单据、盘点、维修、折旧、提醒、留痕的“关联对象”。

## 1) 你在这里要做什么

- 建立资产档案：编号、名称、分类、品牌型号、序列号等
- 维护资产当前状态：在用/闲置/维修/报废
- 维护资产当前归属：仓库/库位/地点、部门、责任人、使用人
- 维护价值信息：原值、税额、净值（净值也可能由折旧产生）
- 维护时间信息：购置日期、入库时间、启用日期、质保到期日

## 2) 字段说明（常用）

| 字段 | 含义 | 建议填写/来源 |
|---|---|---|
| 资产编号（AssetNo） | 唯一编号 | 建议按规则生成（如：IT-2026-0001） |
| 标签码（TagCode） | 扫码内容/二维码内容 | 用于盘点/查找，建议与实物贴标一致 |
| 资产名称（Name） | 资产名称 | 如：戴尔笔记本/投影仪 |
| 分类（CategoryId） | 资产分类 | 从“资产分类”选择 |
| 品牌/型号/规格 | 描述属性 | 便于搜索与采购对账 |
| 序列号（SerialNo） | SN/IMEI | 关键防串货字段 |
| 供应商（VendorId） | 采购/维修供应商 | 从供应商选择 |
| 原值（OriginalValue） | 采购价值 | 财务口径（含税/不含税按你们制度统一） |
| 税额（TaxValue） | 税额 | 可选 |
| 净值（NetBookValue） | 账面净值 | 可选/或由折旧计算 |
| 状态（Status） | 在用/闲置/维修/报废 | 体现资产当前可用性 |
| 仓库/库位/地点 | 存放/使用位置 | 从基础资料选择 |
| 部门/责任人/使用人 | 归属与使用 | 从组织/用户选择 |
| 购置/入库/启用/质保到期 | 关键日期 | 影响提醒与管理动作 |
| 备注（Summary） | 说明 | 如：配置、附件说明等 |

## 3) 典型操作

### A. 新增资产

1. 进入“资产台账”列表，点击“新增资产”
2. 填写资产编号、名称、分类（建议必填）
3. 选择仓库/库位/地点、责任人/使用人
4. 录入价值与关键日期（若需要折旧/质保管理，建议填写）
5. 保存

### B. 编辑资产

常见原因：
- 位置变更（地点/仓库/库位）
- 人员变更（责任人/使用人）
- 状态变更（在用↔闲置↔维修↔报废）

建议：如果你们要求审计留痕，尽量通过“单据/工单/盘点”等业务模块完成变更，而不是直接改台账（这样更可追溯）。

### C. 删除资产

系统一般采用“逻辑删除”（不物理删除），避免历史业务引用断链。

## 4) 与其它模块的关系

- 单据管理：资产的入库/领用/归还/调拨/变更/处置等“业务动作”应产生单据，并可写入留痕
- 盘点计划：盘点对象通常来自资产台账（按范围筛选）
- 维修/保养：工单必须关联资产
- 折旧：折旧配置以资产为维度
- 提醒：质保到期、借用到期等提醒通常基于资产/单据字段
- 资产留痕：台账新增/修改/删除（以及其它业务动作）会写入留痕时间线
